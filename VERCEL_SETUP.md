# Vercel Deployment Setup Guide

## Quick Start (3 Steps)

### Step 1: Create Vercel Postgres Database

1. Go to your Vercel project dashboard
2. Click **Storage** → **Create Database** → **Postgres**
3. Name it `record-db`
4. Copy the connection string (it will be auto-added as `DATABASE_URL`)

### Step 2: Verify Environment Variables

In your Vercel project, go to **Settings** → **Environment Variables** and confirm:

- ✅ `DATABASE_URL` - PostgreSQL connection string (auto-added)
- ✅ `BLOB_READ_WRITE_TOKEN` - Auto-generated by Vercel

### Step 3: Deploy & Run Migrations

1. Push your code to GitHub (or redeploy in Vercel)
2. After deployment succeeds, run migrations:

```bash
# Option A: Using Vercel CLI
vercel env pull
npx prisma migrate deploy

# Option B: In Vercel Deployment Logs
# Go to Deployments → Click latest → Scroll to see if migrations ran
```

## Detailed Setup

### For Vercel Postgres

1. **Create Database:**
   - Vercel Dashboard → Storage → Create Database → Postgres
   - Select region (same as your functions for best performance)
   - Click "Create"

2. **Environment Variables Auto-Added:**
   - `DATABASE_URL` - Automatically added ✓
   - `POSTGRES_PRISMA_URL` - Also added (optional)
   - `POSTGRES_URL_NON_POOLING` - Also added (optional)

### For Vercel Blob Storage

1. **Blob Storage:**
   - Automatically available when you deploy
   - `BLOB_READ_WRITE_TOKEN` is auto-generated
   - No additional setup needed

## Troubleshooting

### Build Fails: "DATABASE_URL not found"

**Cause:** Environment variable not set during build

**Solution:**
1. Go to Vercel project → Settings → Environment Variables
2. Verify `DATABASE_URL` is set
3. Redeploy

### "Failed to fetch recordings: 500"

**Cause:** Migrations not run or database not connected

**Solution:**
```bash
# Pull environment variables
vercel env pull

# Run migrations
npx prisma migrate deploy

# Redeploy
vercel deploy --prod
```

### "BLOB_READ_WRITE_TOKEN not found"

**Solution:**
1. Go to Vercel project → Storage → Blob
2. Create a new Blob store if needed
3. Token will be auto-available

## Local Development

For local development, use SQLite:

```bash
# Install dependencies
npm install

# Start dev server (uses .env.local with SQLite)
npm run dev
```

The app will use `dev.db` locally.

## Environment Variables Reference

| Variable | Local | Vercel | Required |
|----------|-------|--------|----------|
| DATABASE_URL | file:./dev.db | PostgreSQL URL | ✓ |
| BLOB_READ_WRITE_TOKEN | Not needed | Auto-generated | ✓ |
| NODE_ENV | development | production | Auto |

## File Structure

```
.env                    # Placeholder (don't commit secrets)
.env.local             # Local SQLite config (git ignored)
prisma/schema.prisma   # PostgreSQL schema
prisma/migrations/     # Database migrations
```

## Useful Commands

```bash
# View database in browser
npx prisma studio

# Run migrations
npx prisma migrate deploy

# Create new migration
npx prisma migrate dev --name add_feature

# Reset database (dev only)
npx prisma migrate reset

# Check database connection
npx prisma db execute --stdin < /dev/null
```

# Vercel Deployment Setup Guide

## Prerequisites

1. Vercel account (https://vercel.com)
2. GitHub repository connected to Vercel

## Step 1: Set Up Vercel Postgres Database

1. Go to your Vercel project dashboard
2. Click **Storage** → **Create Database** → **Postgres**
3. Name it `record-db` (or any name)
4. Copy the connection string

## Step 2: Set Environment Variables

In your Vercel project settings, go to **Settings** → **Environment Variables** and add:

### Required Variables:

```
DATABASE_URL=<your-postgres-connection-string>
BLOB_READ_WRITE_TOKEN=<auto-generated-by-vercel>
```

**How to get them:**

- **DATABASE_URL**: From Vercel Postgres setup (Step 1)
- **BLOB_READ_WRITE_TOKEN**: Automatically available in Vercel (no setup needed)

### To verify BLOB_READ_WRITE_TOKEN exists:

1. Go to **Settings** → **Environment Variables**
2. You should see `BLOB_READ_WRITE_TOKEN` already listed
3. If not, go to **Storage** → **Blob** and create a new store

## Step 3: Deploy

1. Push your code to GitHub
2. Vercel will automatically deploy
3. The build will run: `prisma generate && prisma migrate deploy && next build`

## Step 4: Verify Deployment

1. Go to your Vercel deployment URL
2. Open browser DevTools (F12)
3. Go to **Console** tab
4. Try recording an audio
5. Check for any errors

## Troubleshooting

### "Failed to fetch recordings: 500"

**Check:**
1. Is `DATABASE_URL` set in Vercel environment variables?
2. Are migrations applied? (Check Vercel build logs)
3. Is the database accessible?

**Solution:**
```bash
# Manually run migrations
npx prisma migrate deploy
```

### "BLOB_READ_WRITE_TOKEN not found"

**Solution:**
1. Go to Vercel project → **Storage** → **Blob**
2. Create a new Blob store if it doesn't exist
3. The token will be automatically available

### Database Connection Error

**Check:**
1. DATABASE_URL is correct (copy from Vercel Postgres)
2. Database is in the same region as your function
3. Try connecting with psql: `psql <DATABASE_URL>`

## Local Development

For local development, the app uses SQLite:

```bash
npm install
npm run dev
```

Database file: `dev.db`

## Production vs Development

| Feature | Local | Vercel |
|---------|-------|--------|
| Database | SQLite (dev.db) | PostgreSQL |
| Storage | Filesystem (public/recordings) | Vercel Blob |
| Detection | `process.env.NODE_ENV` | `process.env.VERCEL === "1"` |

## Useful Commands

```bash
# Check database connection
npx prisma db execute --stdin < /dev/null

# View database
npx prisma studio

# Run migrations
npx prisma migrate deploy

# Reset database (development only)
npx prisma migrate reset
```
